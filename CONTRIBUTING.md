## 最小贡献规则

- 新功能必须补相关文档
- 新配置必须补配置参考文档
- 新 Schema 字段必须补 Schema 示例和参考文档
- 新模块如职责独立，应补本目录 `README.md`

## Git 与 PR 基本要求

- 功能开发通过分支进行，不直接在主分支堆积功能代码
- 使用 PR 合并
- 一个 PR 只完成一件事
- 使用 Conventional Commits
