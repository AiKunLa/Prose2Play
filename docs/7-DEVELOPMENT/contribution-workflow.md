## 标准流程

1. 从主分支切出主题分支
2. 保持一个分支只完成一个明确目标
3. 按小步提交，提交信息遵循 Conventional Commits
4. 本地运行相关检查
5. 发起 PR，并按模板填写完整描述
6. 通过评审后合并

## 提交前检查

- 代码变更是否补文档
- 配置变更是否补参考手册
- Schema 变更是否补示例与参考
- 新模块是否补局部 `README.md`
- 是否运行了相关检查命令
